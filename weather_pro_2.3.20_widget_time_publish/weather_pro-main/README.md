# Weather Pro

**Weather Pro** — Android-приложение погоды без рекламы, рекламных SDK и лишних трекеров.

Текущая версия: **2.3.20**  
`versionCode`: **25**  
`applicationId`: **com.ltm.weather**

## Что реально есть в приложении

- текущая погода по выбранному населённому пункту;
- поиск города вручную;
- определение населённого пункта по геолокации;
- почасовой прогноз, начиная с текущего времени;
- прогноз на 3 / 5 / 7 / 10 дней с сохранением выбранного диапазона;
- дата в строках прогноза в формате день недели + ДД.ММ;
- дневная и ночная температура;
- восход, закат, длина дня;
- UV-индекс, давление, ветер, порывы ветра, облачность и вероятность осадков;
- ночью UV отображается как отсутствующий;
- уведомления о вероятности дождя по выбранному порогу;
- настройка “Погода на экране блокировки / AOD” через постоянное тихое уведомление;
- светлая / тёмная / системная тема приложения;
- исправлена читаемость status bar/navigation bar в тёмной теме;
- светлая / тёмная / системная тема виджета;
- выбор языка и единиц измерения;
- улучшенная реалистичная погодная анимация на главном экране: солнце, луна, облака, дождь, снег, туман и гроза;
- виджет “Часы + погода” 5×2;
- виджет “Часы по центру + погода” 5×2;
- виджет почасового прогноза;
- виджет прогноза на 3 дня;
- настройка прозрачности виджетов;
- настройка размера часов и погодного блока;
- отображение населённого пункта на виджете без улицы и координат;
- отображение фазы луны на виджете ночью;
- реалистичные погодные изображения на основных виджетах вместо системных эмодзи;
- увеличенные погодные изображения на виджетах “Часы + погода” и “Часы по центру + погода”;
- системная синхронизация времени на виджетах через `TextClock`: часы берут время напрямую из Android и обновляются вместе с временем телефона;
- увеличенная кнопка-шестерёнка на виджетах для более удобного попадания;
- стабильная фаза луны по дате;
- предпросмотры в выборе виджетов используют совместимые `previewLayout` и запасные непрозрачные PNG `previewImage`, чтобы выглядеть как реальные виджеты и иметь fallback для разных лаунчеров;
- живой предпросмотр виджета прямо в настройках приложения;
- повторная настройка уже установленного виджета по маленькой кнопке-шестерёнке на самом виджете;
- возврат на рабочий стол после повторной настройки виджета;
- обновление виджетов после перезагрузки, обновления приложения, смены времени, смены часового пояса и разблокировки экрана;
- публикационный комплект: GitHub Actions собирает подписанные Release APK и Release AAB, fastlane metadata и changelog обновлены под текущую версию;
- совместимость с Android 10+ и разными оболочками/лаунчерами насколько это возможно через стандартные Android API.


## Источники данных

- прогноз погоды: Open-Meteo;
- поиск города: Open-Meteo Geocoding;
- обратное определение населённого пункта: Android Geocoder, резервно OpenStreetMap Nominatim.

Приложение не использует API-ключ для Open-Meteo.

## Сборка через GitHub Actions

1. Загрузи проект в репозиторий GitHub.
2. Открой вкладку **Actions**.
3. Запусти workflow **Build Weather Pro Android**.
4. После сборки скачай артефакт **Weather-Pro-builds**.

Собираются:

- Debug APK;
- signed Release APK;
- signed Release AAB.

## Обновление поверх установленной версии

Для обновления поверх старой версии нужно сохранять:

- тот же `applicationId`: `com.ltm.weather`;
- тот же ключ подписи;
- увеличенный `versionCode`.

В проекте есть стабильный тестовый ключ:

`app/dev-update-key.jks`

Debug и release-сборки подписываются через `signingConfigs.updateCompatible`, чтобы APK из этого проекта могли обновляться поверх друг друга.

Если на телефоне уже установлена версия с тем же `applicationId`, но с другой подписью, Android покажет конфликт пакетов. Такую старую версию невозможно обновить поверх без старого ключа подписи — её нужно удалить один раз, поставить сборку из этого проекта, и дальше новые версии будут ставиться поверх.

Для публикации в магазине лучше использовать свой постоянный release/upload key через GitHub Secrets и больше его не менять.

Поддерживаемые secrets:

- `WEATHER_KEYSTORE_BASE64` / `LTM_KEYSTORE_BASE64`;
- `WEATHER_KEYSTORE_PASSWORD` / `LTM_KEYSTORE_PASSWORD`;
- `WEATHER_KEY_ALIAS` / `LTM_KEY_ALIAS`;
- `WEATHER_KEY_PASSWORD` / `LTM_KEY_PASSWORD`.

Дополнительно для публикации подготовлены:

- `fastlane/metadata/android/ru-RU/`;
- `fastlane/metadata/android/ru-RU/changelogs/25.txt`;
- `CHANGELOG.md`;
- `PUBLICATION_CHECKLIST.md`.

## Разрешения

Приложение может запрашивать:

- интернет — для получения прогноза;
- геолокацию — для определения текущего населённого пункта;
- уведомления — для уведомлений о дожде и погоды на экране блокировки / AOD;
- автозапуск после перезагрузки — для восстановления обновлений виджетов и уведомления погоды.

## Локальная сборка

```bash
./scripts/build-debug.sh
./scripts/build-release.sh
```

Для Windows:

```powershell
.\scripts\build-debug.ps1
.\scripts\build-release.ps1
```

## Примечание по виджетам

Android-виджеты зависят от лаунчера. Разные оболочки Vivo, Honor, TCL, Xiaomi, Samsung и другие могут по-разному обрабатывать клики, предпросмотр и обновления. В проекте используются стандартные совместимые механизмы Android: `AppWidgetProvider`, `AlarmManager`, `updatePeriodMillis`, максимально простые layout-превью, запасные PNG-предпросмотры и стабильные PendingIntent.
